# Подземелья и песики — многопользовательская карточная игра

Клиент-серверная игра на Java, где игроки исследуют подземелье, выкладывая карты комнат, сражаются с врагами, собирают монеты и артефакты, а затем пытаются выйти на поверхность, проложив путь через лабиринт.

**Архитектура**: TCP-сокеты, многопоточный сервер с игровой логикой, JavaFX-клиент с отрисовкой поля и анимациями.

---

## Особенности

- **Чистый Java без фреймворков** (Sockets, JavaFX, многопоточность)
- **Полноценный игровой цикл**: 3 раунда, подсчёт монет, состояния игроков (спит / вышел)
- **Система карт**: комнаты загружаются из CSV, имеют выходы (стороны), тип события (враг, монета, артефакт, старт)
- **Сетевое взаимодействие**: асинхронные сообщения через `BlockingQueue`, свой протокол
- **Клиент на JavaFX**:
  - Отрисовка доски (сетка, карты, доступные позиции)
  - Анимация поворота карт перед размещением
  - Выбор пути выхода кликами по комнатам
  - Модальное окно не нужно, но всё интерактивно
- **Серверная логика**:
  - Проверка возможности разместить карту (совпадение выходов)
  - Валидация пути выхода (соседние комнаты, наличие выхода на границе)
  - Управление токенами (ресурс для прохода через комнаты)
  - Окончание раунда, когда все игроки вышли или уснули

---

## Технологии

- Java 11+ (JavaFX отдельно или bundled в 11+)
- JavaFX (графический клиент)
- Сокеты (TCP/IP)
- Многопоточность (Thread, ExecutorService)
- Свой игровой движок (без сторонних библиотек, кроме JavaFX и стандартных)

## Как запустить

### Требования
- JDK 11 или новее (лучше 17)
- Для JavaFX: если JDK не содержит JavaFX (например, Oracle JDK 11+), нужно добавить модули. **Убедитесь, что JavaFX установлена**. Проще всего использовать **IntelliJ IDEA** с плагином JavaFX.

### Запуск сервера

1. Склонировать репозиторий:
   ```bash
   git clone https://github.com/Umarrina/dungeons-and-doggies.git
   cd dungeons-and-doggies
   ```
2. Открыть проект в IDEA.
3. Найти класс `ru.kpfu.itis.group400.amirova.server.GameServer` и запустить его.
    - Можно указать аргументы (порт, хост, макс. игроков). По умолчанию: `localhost:5555`, макс. 2 игрока.
    - После запуска сервер ждёт подключений.

### Запуск клиента

1. Запустить класс `ru.kpfu.itis.group400.amirova.client.GameClient` (JavaFX приложение).
2. В появившемся диалоге ввести:
    - Хост: `localhost`
    - Порт: `5555`
    - Имя игрока (например, «Арина»)
3. Подключиться. Когда подключатся два игрока, игра начнётся автоматически.

### Альтернативный запуск через командную строку (если настроены модули)

```bash
# Для клиента (пример для Linux/Mac, путь к JavaFX)
java --module-path /path/to/javafx-sdk/lib --add-modules javafx.controls,javafx.fxml -cp target/classes ru.kpfu.itis.group400.amirova.client.GameClient
```

> **Примечание**: `configuration.csv` должна лежать в classpath (в `resources`). В проекте она уже есть.

## Как играть (краткие правила)

### Цель
Набрать как можно больше монет за 3 раунда. В конце раунда игроки, вышедшие из подземелья, получают монеты с собранных по пути комнат-монет и врагов.

### Ход игрока
- **Исследовать** – взять новую карту комнаты из колоды и разместить её на поле рядом с уже выложенными комнатами (выходы должны совпадать).
- **Повернуть** – вращать карту на руке перед размещением (меняет направления выходов).
- **Выйти** – выбрать путь от стартовой комнаты до любой границы поля. За каждую комнату на пути нужно заплатить 1 токен (токены даются в начале раунда). За комнаты с монетами/врагами получаешь монеты.
- **Спать** – пропустить ход, получить +1 к макс. токенам и восстановить 1 токен. Нельзя спать с картой на руке.

### Окончание раунда
Когда все игроки вышли или уснули, раунд заканчивается. Монеты суммируются. Через 3 раунда побеждает игрок с наибольшим числом монет.

## Структура проекта (основные пакеты)

```
src/main/java/ru/kpfu/itis/group400/amirova/
├── client/
│   ├── GameClient.java           # JavaFX-клиент, подключение, обработка сообщений
│   └── ui/
│       ├── GameUI.java           # Отрисовка поля, карт, кнопки, события

│   ├── GameServer.java           # Сервер, принятие подключений, ClientHandler

│   ├── GameSender.java           # Утилиты для отправки сообщений игрокам
│   ├── GameSerializer.java       # Парсинг позиций и т.п.
│   └── game/                     # Игровая логика
│       ├── GameEngine.java       # Основные действия: разместить, повернуть, выйти

│       ├── GameInitializer.java  # Загрузка карт и собак из CSV
│       ├── GameSetupManager.java # Начальная инициализация игроков
│       ├── model/                # Сущности (игрок, комната, позиция, состояние)
│       └── ...
└── resources/
    └── configuration.csv         # База данных карт (комнат)
```
